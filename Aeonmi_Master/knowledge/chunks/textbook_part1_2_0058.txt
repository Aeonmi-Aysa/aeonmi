=== The Descriptor Format ===

BODY:
The key to understanding quantum_run() is the circuit descriptor format. Rather than building circuit objects programmatically, Aeonmi uses a compact string descriptor that encodes the complete quantum circuit specification.

The descriptor format is:

CODE:
```
// Descriptor format:
// "n_q n_c shots op_count  [op_type target control] x op_count"
//
// n_q       = number of qubits
// n_c       = number of classical bits (measurement results)
// shots     = number of times to run the circuit
// op_count  = number of operations
//
// Each operation is three numbers: type, target, control
//   type:    0=H, 1=X, 2=Y, 3=Z, 4=CX/CNOT, 5=S, 6=T, 7=MEASURE
//   target:  qubit index (0-based)
//   control: qubit index for two-qubit gates, -1 for single-qubit gates

// Example: Bell state circuit
// H on qubit 0, CX with ctrl=0 target=1, MEASURE qubit 0, MEASURE qubit 1
let bell_descriptor = "2 2 512 4  0 0 -1  4 1 0  7 0 0  7 1 1";
//                     ^^ n_q=2 qubits
//                        ^^ n_c=2 classical bits
//                           ^^^ shots=512
//                               ^ op_count=4
//                                  ^ H(type=0) ^ target=0  ^control=-1
//                                              ^ CX(type=4) ^ target=1 ^ ctrl=0
//                                                            ^ MEASURE(type=7) q0
//                                                                           ^ MEASURE q1
```
