=== The Philosophy ===

BODY:
Aeonmi is an AI-native programming language built from the ground up in Rust. It is not a scripting language ported to a new runtime, not a transpiler to JavaScript, and not a thin wrapper around Python. Aeonmi is a complete language ecosystem: its own lexer, parser, abstract syntax tree, intermediate representation, and virtual machine — all written natively in Rust, compiling to zero JavaScript, generating zero interpreted bytecode from an external platform.

The core design principle of Aeonmi is stated plainly in its roadmap: Built by AI for AI. This is not a marketing tagline. It is a load-bearing architectural constraint. It means that the language was designed so that an AI system — specifically Mother AI, the cognitive system at the center of the Aeonmi ecosystem — can author, read, execute, and reason about Aeonmi programs as a first-class activity. The AI is not a user of a tool built for humans. The AI is both builder and primary consumer.

This distinction matters for how you learn the language. Aeonmi syntax is designed to be unambiguous, regular, and parseable by a system that must understand what it writes. Function declarations are explicit. Types are inferable but unambiguous. The quantum layer uses well-defined descriptor strings rather than a complex API that would be difficult for a generative system to produce correctly.

BODY:
The language has a dual identity. For human programmers, Aeonmi looks and feels like a familiar imperative scripting language with some distinctive additions: quantum operations, f-string interpolation, a rich standard library, and a self-hosting compiler written in its own syntax. For AI systems, Aeonmi is infrastructure — the substrate on which a cognitive architecture can grow, propose programs, build them, and reflect on the results.

NOTE: Aeonmi is not finished. The roadmap is honest about this. The self-hosting compiler (called the Shard) has its bootstrap pipeline running but does not yet fully compile itself. The IBM quantum hardware bridge works in code but requires a live IBM token for real-device execution. The .ai operational modules run standalone but are not yet wired into the Mother AI cognitive layer at runtime. This textbook teaches the language as it actually runs today, not as it will run in future phases.
