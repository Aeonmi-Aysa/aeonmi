=== genesis_sync.py — The Synchronization Bridge ===

BODY: genesis_sync.py is the Python script that ensures genesis.json stays
coherent across Rust writes and Python reads. It runs when called via
`/api/sync` from the dashboard, or when `sync` is run in the REPL.

The sync script reads the current genesis.json, merges the Python operational
data with the Rust cognitive data, validates the schema version, and writes a
unified file. It preserves all three tracks — it does not overwrite the
cognitive section with Python data or vice versa. It is a merge, not a replace.

The risk with concurrent writes: if the Rust REPL and the Python dashboard both
attempt to write genesis.json within the same second, one write will silently
overwrite the other's data. There is no file lock. This is acceptable for
single-user operation but is a known risk in concurrent scenarios.

CODE:
```
// Calling genesis_sync from REPL:
> sync
// Forces immediate sync without waiting for 5-interaction threshold

// Calling genesis_sync from the dashboard:
// POST to /api/sync
// Response: { "status": "ok", "schema_version": "5.0", "cognitive": {...}, ... }

// Manual sync from command line:
python Aeonmi_Master/genesis_sync.py
// Reads, merges, validates, writes genesis.json
```
